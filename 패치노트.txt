#  v1.0.0  —  2026-06-13 (토)   "정식 출시"
##################################################################

  드디어 기일 일정 알림 봇이 세상에 나왔습니다!
  매일 직원이 Lawware에 입력한 기일을 Discord로 자동 정리해 쏴줍니다.

  [NEW] 일정 자동 알림 봇 최초 가동
        - Google Calendar(서비스계정·읽기전용)에서 하루치 일정을 읽어
          회사 표준 양식으로 변환 후 Discord 웹훅으로 전송.
  [NEW] 여러 캘린더 동시 지원
        - 캘린더 ID를 쉼표/공백으로 여러 개 입력 가능.
        - 구글 계정이 2개로 흩어져 있어도 계정별 키(_2, _3…)로 모두 수집.
  [NEW] 회사 표준 출력 양식 확립
        - "YYMMDD 요일" 헤더 + "시간 [의뢰인] 기일종류 > 출석변호사" 한 줄 양식.
        - 정식 기일(사건번호+내용 보유)은 상세 양식, 그 외 일정은 제목 그대로.
        - 조사기일은 의뢰인 연락처까지 아랫줄에 표기.
        - 여러 캘린더에 겹쳐 들어온 동일 일정은 자동 중복 제거.
  [개선] 헤더 디자인 정리
        - 시간 미특정(종일) 일정을 맨 위로 모아 배치.
        - 헤더 접두를 '#' → 캘린더 이모지(📅)로 변경.
  [NEW] 오후 '익일 송무팀별' 알림 추가
        - 오후에 내일 일정을 송무1/2/3팀 + 상담지원팀으로 분류해 따로 발송.
        - 팀별 메시지 사이는 보이지 않는 구분선(zero-width space)으로 깔끔하게.


##################################################################
#  v1.1.0  —  2026-06-14 (일)   "제 시간에 도착합니다"
##################################################################

  "알림이 왜 늦게 와요?" 문제를 정조준한 정시성 패치.

  [운영] 정시 발송 체계로 전환
        - 원인: GitHub Actions의 schedule cron은 정시 실행을 보장하지 않음
          (실제 07:00→08:11, 17:45→20:17 지연 발생).
        - 해결: 외부 무료 스케줄러(cron-job.org)가 정시에 GitHub를 깨우도록 변경.
          GitHub 자체 cron은 중복 발송 방지를 위해 비활성화(주석 처리).
        - 결과: 큐 혼잡과 무관하게 정시 도착, 지연 거의 소멸.
  [문서] 운영 문서 정비
        - '전체 구성도', '작업 기록', '외부 스케줄러 설정 가이드' 신설.
        - README에서 docs 링크 연결.
  [운영] 로컬 실행 편의 개선
        - python-dotenv 추가 → 로컬에서 .env 자동 로딩.

#  v1.4.0  —  2026-06-18 (목)   "비고 노출 · 선고 대리 일원화"   ★최신★
##################################################################

  캘린더 '비고'란을 알림에 끌어오고, 선고기일 대리출석 표기를 정리.

  [NEW] '비고' 선별 노출
        - 비고 내용이 '장소' 또는 '제출/이행 여부'일 때만 아랫줄에
          원문 그대로(라벨 없이) 표시. 그 외(복대리·메모 등)는 숨김.
        - 제출 키워드(제출·접수·발송·납부·이행·체결·완료·미제출 등),
          장소 키워드(법원·검찰청·경찰서·법정·조정실 등)·전화번호로 판별.
          두 키워드가 함께 있으면 '제출'을 우선.
        - 장소성 비고는 기존 '장소' 줄을 덮지 않고 함께 병기(둘 다 표기).
          (예: 경찰서 장소 + 비고의 검찰청 조정실을 나란히 노출)
        - 판별 키워드는 transform.py 의 BIGO_SUBMISSION_KW /
          BIGO_LOCATION_KW 표만 고치면 운영 중 조정 가능.
  [개선] 선고기일 대리출석 표기 일원화
        - 선고기일에 한해 출석변호사의 '복대리·청취대리·선고청취대리·
          선고청취' 표기를 모두 '청취대리'로 통일.
        - 다른 기일종류(변론·공판 등)의 표기는 원문 그대로 유지.
  [문서] 회귀테스트 6건 추가(비고 분류·병기·선고 일원화), README 규칙 반영.
