## 2026-06-14 — 알림 지연 해결: 외부 스케줄러로 전환

### 문제
- 오전 알림(예약 07:00)이 **08:11**에, 오후 알림(예약 17:45)이 **20:17**에 도착. 정시에 안 옴.

### 원인
- 코드/설정 문제가 아님. **GitHub Actions의 `schedule:` cron은 정시 실행을 보장하지 않음**(공식 best-effort).
  전 세계 작업이 정시에 몰리면 큐가 밀려 수십 분~수 시간 지연됨.

### 해결 (선택: 외부 스케줄러 트리거)
1. **외부 무료 스케줄러 `cron-job.org`** 가 매일 정시(KST 07:00 / 17:45)에
   GitHub API `workflow_dispatch` 를 호출해 워크플로를 깨우도록 변경.
   - 외부 서비스는 GitHub 큐와 무관하게 정시 동작 → 지연 거의 사라짐.
2. **중복 발송 방지**: 두 워크플로(`daily.yml`, `evening.yml`)의 GitHub `schedule:` cron을
   **주석 처리(비활성화)**. `workflow_dispatch` 만 남김.
3. 설정 가이드 문서 작성: [`외부-스케줄러-설정.md`](./외부-스케줄러-설정.md)
4. 전체 구성도 작성: [`전체-구성도.md`](./전체-구성도.md)

관련 커밋: `a422959` (schedule cron 비활성화 + 외부 스케줄러 트리거로 전환)

### 트러블슈팅 기록 — cron-job.org 설정 중 404
- 증상: cron-job.org 작업 결과에 `{"message":"Not Found","status":"404"}`.
- 원인: **URL이 아니라 토큰(인증) 문제.** GitHub은 토큰이 리소스 접근 권한이 없으면
  403이 아닌 **404** 를 돌려줌(존재 여부 은폐 목적).
- 확인한 사실:
  - 저장소 `nova-law-legal/legal` = 공개, 기본 브랜치 `main`.
  - 워크플로 `daily.yml`(id 295211411) / `evening.yml`(id 295253972) 모두 active. → **URL 정상.**
- 해결: Fine-grained PAT의 권한을 **Actions: Read and write** + 대상 저장소 포함으로 맞추고,
  `Authorization: Bearer <토큰>` 헤더를 정확히 설정.
- 검증: 토큰 직접 테스트 시 응답 헤더에 `X-RateLimit-Limit: 5000`(인증됨),
  `x-accepted-github-permissions: actions=write`, 토큰 만료 2027-06-13 확인 → 정상.
- 결과: **두 워크플로 모두 `workflow_dispatch`로 실행 성공**(2026-06-14 13:05, 13:06 KST).

### cron-job.org 비용
- **완전 무료.** 신용카드 불필요. 하루 2회 호출은 무료 한도 내.
- 무료 제약: 최소 간격 1분(무관), 요청 제한시간 ~30초(우리는 트리거만 보내고 끝나 무관), 유료 SLA 없음.
- 보안: 이 서비스에 GitHub 토큰이 저장되므로 토큰 권한은 최소(저장소 1개 + Actions write)로 유지.

## 현재 운영 상태 (2026-06-14 기준)

| 항목 | 상태 |
|---|---|
| 오전 알림 | `daily.yml`, 외부 트리거 KST 07:00, 오늘 일정 전체 → 기본 채널 |
| 오후 알림 | `evening.yml`, 외부 트리거 KST 17:45, 내일 일정 → 팀별 채널 |
| GitHub `schedule:` cron | **비활성화(주석)** — 외부 트리거 단독 운영 |
| 트리거 수단 | cron-job.org (무료) → GitHub API `workflow_dispatch` (Fine-grained PAT) |
| 검증 | 수동 dispatch 실행 성공 확인 완료 |

## 나중에 확인/할 일 체크리스트

- [ ] cron-job.org 두 작업의 **타임존이 Asia/Seoul(KST)** 인지 확인 (UTC면 9시간 어긋남)
- [ ] 실제 스케줄 시각(07:00 / 17:45)에 정시 도착하는지 며칠 관찰
- [ ] GitHub PAT 만료일(**2027-06-13**) 다가오면 갱신하고 cron-job.org 헤더 토큰 교체
- [ ] 외부 트리거가 죽는 만일에 대비하려면, 워크플로의 주석된 `schedule:` 을 백업으로 되살릴 수 있음
